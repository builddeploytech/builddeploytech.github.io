CI/CD Pipeline using GitHub & Jenkins

By BuildDeploy Tech

CI/CD (Continuous Integration and Continuous Deployment) helps teams deliver software faster and more reliably by automating build, test and deployment processes.

What is CI/CD?

Continuous Integration ensures every code change is automatically built and tested, while Continuous Deployment automates releasing code to production.

Step 1: Setup Jenkins Server

Install Jenkins on a Linux server: 


sudo apt update
sudo apt install openjdk-17-jdk -y
sudo apt install jenkins -y

Step 2: Connect GitHub Repository

Create a new Jenkins job and connect your GitHub repository using credentials.

Step 3: Configure Webhooks

Add GitHub webhook so that Jenkins pipeline triggers automatically on code push.

Step 4: Jenkins Pipeline

Sample Jenkinsfile: 


pipeline {
  agent any
  stages {
    stage('Build') {
      steps {
        echo 'Building project'
      }
    }
    stage('Deploy') {
      steps {
        echo 'Deploying application'
      }
    }
  }
}

Benefits of CI/CD

  • Faster releases
  • Reduced manual errors
  • Better code quality
  • Improved team productivity
